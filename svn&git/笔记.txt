svn工具（不超过1层）：集中式版本控制管理工具
	 什么公司会用svn：
		1、老公司
		2、项目是保密的！！！
		
	 服务器端：创建一个svn项目仓库
	   1、创建用户（给我的员工创建）
	   2、创建仓库（用于放我们的项目）很多小操作（比如分配权限）

	 客户端：检出项目仓库
	   1、安装svn客户端
	   2、拿着项目经理给你的地址，任意位置右键svn checkout（检出）
		将仓库拉到我们的电脑
	      里面就可以去建立自己的文件夹，写项目了
	   3、写完后，右键TortoiseSVN，点击add，全部添加，添加到了本地svn
	   4、右键SVN commit 提交到服务器端

	   别的人想看见，需要一个操作
		右键 SVN UPDATE

	svn工具：最终所有的东西都是保存在服务器端的，客户端随时随地可拉

	缺点：集中式：万一服务器坏了
	建议：
	      早上来了先更新
	      中午就提交
	      下班就提交
	        人不在一定要锁电脑

git工具（9层以上）：
	分布式版本控制管理工具，一台电脑既可以是客户端也可以是服务器端（云端）
	1、注册github账号，并且登录
	2、github创建一个仓库

	3、安装git客户端：
		先安装：Git-2.18.0-64-bit.exe
		在安装：TortoiseGit-2.6.0.0-64bit.msi
	面试题：*4、将云端的仓库拉倒本地来的命令：
		   git clone 仓库路径

		*5、将仓库的内容上传到云端
		   1、git add .
		   2、git commit -m"必须写日志"
		   3、git push -v origin master

		*6、线上的内容更新下来，一般拉下来都用他
		   git pull origin master（或者重新克隆）
	
	7、如果你希望某些文件不会提交到github
	   创建一个.gitignore的文件
